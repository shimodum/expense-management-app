# 画面一覧

本ドキュメントは、`01_requirements.md` の要件を踏まえ、経費精算アプリのMVPで用意する画面を整理したものである。実装(ルーティング・コントローラ・View設計)の前段として、画面単位で目的・操作・アクセス条件・遷移を明確にすることを目的とする。

対象外機能(通知、集計ダッシュボード、CSV出力、管理者による経費申請登録、多段階承認)に関する画面はMVPの対象外とし、本一覧には含めない。

## 設計方針の要点

- 一般社員と管理者でログイン後の初期画面を分ける(SC-02 / SC-05)。
- 経費申請の登録画面と編集画面は同一フォーム(SC-04)を共用する。
- 提出操作は編集画面ではなく詳細画面(SC-03)で行う(確認モーダル経由)。
- 管理者の承認・却下は詳細画面(SC-06)内で完結させる(専用画面は設けない)。
- 管理者は提出済みだけでなく、承認済み・却下済みの申請も一覧・詳細で閲覧できる(管理対象は提出済み・承認済み・却下の3ステータスであり、未提出の下書きは管理者の閲覧対象外とする)。
- 削除・提出・再提出・却下理由コメント入力は、独立画面ではなくモーダルで対応する。

## モーダルで完結する操作(独立画面にしない)

以下の操作は、詳細画面上のモーダルダイアログで完結させ、独立した画面は設けない。

| 操作 | 対象画面 | 対象ステータス | 補足 |
|---|---|---|---|
| 削除確認 | SC-03 | 下書きのみ | 削除は下書き状態の申請にのみ許可 |
| 提出確認 | SC-03 | 下書き→提出済み | 内容確認後に提出することを想定した確認モーダル |
| 再提出確認 | SC-03 | 却下→提出済み | SC-04での編集・保存後もステータスは「却下」のまま維持されるため、SC-03で修正内容を確認したうえで再提出ボタンから確認モーダルを表示し、確認後にステータスを「却下」から「提出済み」へ変更する |
| 却下理由コメント入力 | SC-06 | 提出済み→却下 | 却下理由は任意項目 |

## 画面遷移の補足

- ログイン(SC-01)後、一般社員はSC-02、管理者はSC-05へ遷移する(ロール別の初期画面)。
- 一般社員の主な流れ: SC-02(一覧) → SC-04(新規登録、下書き保存) → SC-03(詳細) → 提出モーダル → 提出済みへ。却下時はSC-03 → SC-04(編集、保存してもステータスは却下のまま) → SC-03(詳細へ戻り修正内容を確認) → 再提出モーダル → 提出済みへ遷移。
- 管理者の主な流れ: SC-05(一覧、提出済みで絞り込み) → SC-06(詳細) → 承認 or 却下(コメント入力モーダル) → SC-05へ戻る。
- ログアウトは画面ではなく、ヘッダー等に配置する操作として扱う(全画面共通)。
